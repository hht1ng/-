# 表结构数据 Skill

## 状态：非必选（文档提供时读取，未提供则跳过）

该部分是实际数据表结构的**补充兜底参考**，当指标或维度在指标维度说明书和数据字典中无法匹配时使用。同时可用于过滤条件的字段名校验。

## 数据位置

数据位于用户上传的 `表结构数据` 独立文档中。

## 文档结构

表结构数据支持两种输入格式，两种格式可在同一文档中混用（不同 Sheet 页或不同数据区域）：

### 格式一：DDL 格式

直接提供表的 CREATE TABLE 语句，包含完整字段名和字段类型。

| 列 | 用途 |
|----|------|
| 表名 | 数据库表名称 |
| 表创建语句 | 完整的 CREATE TABLE DDL 语句 |

### 格式二：字段级格式

逐字段列出表结构详情，包含字段注释。

| 列 | 用途 |
|----|------|
| 表名 | 数据库表名称 |
| 表注释 | 表的业务含义说明 |
| 字段名称 | 表中的字段名称 |
| 字段注释 | 字段的业务含义说明 |
| 数据类型 | 字段的数据类型 |

## 读取策略

1. **检查文档是否提供**：检查用户是否上传了 `表结构数据` 文档
2. **文档已提供**：
   - 识别每个数据区域的格式（DDL 格式或字段级格式）
   - DDL 格式：解析 CREATE TABLE 语句，提取 `字段名 → 字段类型` 映射，结合表名构建 `表名 → 字段名 → 字段类型` 映射
   - 字段级格式：直接构建 `表名 → 字段名 → {字段注释, 数据类型}` 映射
3. **文档未提供**：跳过

## 使用方式

### 1. 指标兜底匹配
- 当问句中的指标在指标维度说明书和数据字典中均无法匹配时
- 在表结构数据中搜索字段名和字段注释
- 字段注释中可能隐含指标的业务含义
- 匹配到候选后标记为"需确认指标"，注明来源为表结构数据

### 2. 维度兜底匹配
- 当问句中的维度在指标维度说明书和数据字典中均无法匹配时
- 在表结构数据中搜索相关字段名和字段注释
- 匹配到候选后标记为"需确认维度"，注明来源为表结构数据

### 3. 过滤条件字段校验
- 从问句提取过滤条件后，在表结构数据中验证字段名是否存在
- 若字段名在数据字典和表结构数据中均不存在，标记为"需确认过滤条件"
- DDL 格式可额外提供字段类型信息，辅助判断过滤条件的值格式是否匹配

## 匹配优先级

```
指标维度说明书                       ← 第一优先级
       │（未匹配到）
       ▼
数据字典                            ← 第二优先级（兜底）
       │（未匹配到）
       ▼
表结构数据                          ← 第三优先级（兜底）
       │（未匹配到）
       ▼
从问句文本直接提取 → 标记"需确认"    ← 第四优先级
```

## 读取流程

```
表结构数据（独立文档）
    │
    ├── 检查文档是否提供
    │       ├── 已提供 → 读取，识别格式：
    │       │         · DDL 格式：解析 CREATE TABLE 语句
    │       │         · 字段级格式：直接构建映射
    │       │         → 构建统一的 表名→字段名→详情 映射表
    │       └── 未提供 → 跳过
    │
    └── 继续后续处理（兜底匹配和字段校验时使用）
```

## 两种格式示例

### DDL 格式示例

| 表名 | 表创建语句 |
|------|-----------|
| ods.xxx_table | create table ods.xxx_table( occur_date String, fund_acc_no String, ... ) |

### 字段级格式示例

| 表名 | 表注释 | 字段名称 | 字段注释 | 数据类型 |
|------|--------|---------|---------|---------|
| ast_crd_acc_stock | 信用账户场内持仓表 | occur_date | 交易日 | date |
| ast_crd_acc_stock | 信用账户场内持仓表 | fund_acc_no | 资金账号 | character varying |
