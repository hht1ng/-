# 数据字典 Skill

## 状态：非必选（文档提供时读取，未提供则跳过）

该部分是数据库表和字段的**兜底参考**，当指标或维度在指标维度说明书中无法匹配时使用。

## 数据位置

数据位于用户上传的 `数据字典` 独立文档中。

## 文档结构

| 列 | 用途 |
|----|------|
| 表名 | 数据库表名称 |
| 表说明 | 表的业务含义 |
| 字段名 | 表中的字段名称 |
| 字段类型 | 数据类型 |
| 字段说明 | 字段的业务含义 |
| 备注 | 额外说明 |

## 读取策略

1. **检查文档是否提供**：检查用户是否上传了 `数据字典` 文档
2. **文档已提供**：读取并构建 `表名 → 字段名 → 字段说明` 映射
3. **文档未提供**：跳过

## 使用方式

### 1. 指标兜底匹配
- 当问句中的指标在指标维度说明书中无法匹配时
- 在数据字典中搜索相关字段名和字段说明
- 匹配到候选后标记为"需确认指标"，注明来源为数据字典

### 2. 维度兜底匹配
- 当问句中的维度在指标维度说明书中无法匹配时
- 在数据字典中搜索相关维度字段
- 匹配到候选后标记为"需确认维度"，注明来源为数据字典

### 3. 过滤条件字段校验
- 从问句提取过滤条件后，在数据字典中验证字段名是否存在
- 若不存在，标记为"需确认过滤条件"

## 匹配优先级

```
指标维度说明书                       ← 第一优先级
       │（未匹配到）
       ▼
数据字典                            ← 第二优先级（兜底）
       │（未匹配到）
       ▼
表结构数据                          ← 第三优先级（兜底）
       │（未匹配到）
       ▼
从问句文本直接提取 → 标记"需确认"    ← 第四优先级
```

## 读取流程

```
数据字典（独立文档）
    │
    ├── 检查文档是否提供
    │       ├── 已提供 → 读取，构建字段映射表
    │       └── 未提供 → 跳过
    │
    └── 继续后续处理
```
