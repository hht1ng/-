# 问句转SQL执行指南 (Question-to-SQL)

本技能旨在实现从“自然语言问句” $\rightarrow$ “SQL语句” $\rightarrow$ “执行结果” 的全链路自动化验证。

## 工作流程

### 0. 环境准备
在执行 SQL 之前，必须确保 Python 环境中安装了必要的数据库驱动。
- 执行命令：`pip install sqlalchemy pymysql openpyxl`

### 1. 数据准备与分析
- **读取输入文件**：读取 `问句模板.xlsx` 和 `数据字典.xlsx`。
- **解析元数据（鲁棒性读取）**：
    - **Sheet 匹配原则**：由于模板版本可能不同（如 Sheet 名称包含编号 `01-问句` 或直接命名为 `问句`），**必须先获取所有 Sheet 名称列表，通过关键词模糊匹配定位目标页签**。
    - **指标提取**：在包含 `指标` 关键字的 sheet 中提取指标的计算逻辑和来源表/字段。
    - **维度提取**：在包含 `维度` 关键字的 sheet 中提取维度的来源表/字段。
    - **问句提取**：在包含 `问句` 关键字的 sheet 中提取待转换的自然语言问句。
    - **配置提取**：在包含 `数据来源系统` 关键字的 sheet 中获取数据库连接配置（服务器、端口、账号密码等）。
    - **结构确认**：从 `数据字典.xlsx` 中确认表的物理结构（字段名、类型）。

### 2. SQL 生成 (AI 核心步骤)
针对提取到的每一个问句，按照以下逻辑生成 SQL：
- **识别指标**：匹配问句中的指标，查找其计算逻辑（如 `SUM`, `COUNT(DISTINCT)`, `MAX` 等）。
- **识别维度与过滤条件**：匹配问句中的维度和过滤条件（如 `日期 = '2024-01-01'`, `客户类型 = '机构'`）。
- **构建查询**：
    - 确定主表（事实表）。
    - 根据维度需求进行 `JOIN` 操作。
    - 编写 `WHERE` 子句实现过滤。
    - 编写 `GROUP BY` 实现聚合。
    - 处理 `TOP N` 或 `排序` 需求。
- **命名规范（强制）**：**必须严格按照《数据字典》中定义的英文表名和英文字段名编写 SQL，禁止使用中文名称或自行猜测字段名。**
- **语法标准**：生成的 SQL 必须是标准 SQL。**绝对禁止**在语句开头添加数据库前缀（例如禁止出现 `dbname.SELECT...`）。
- **校验**：确保字段名与 `数据字典` 完全一致。

### 3. 执行与结果输出
使用配套脚本 `scripts/execute_sql_to_excel.py` 完成执行。

**操作步骤**：
1. 将生成的 SQL 列表和数据库配置构建为一个 JSON 文件（例如 `query_config.json`），格式如下：
   ```json
   {
     "db_config": {
       "user": "username",
       "password": "password",
       "host": "host",
       "port": "port",
       "dbname": "dbname"
     },
     "queries": [
       { "question": "问句1", "sql": "SELECT ..." },
       { "question": "问句2", "sql": "SELECT ..." }
     ]
   }
   ```
2. 运行脚本（包含环境准备）：
   ```bash
   pip install sqlalchemy pymysql openpyxl && python skills/question-to-sql/scripts/execute_sql_to_excel.py data/query_config.json
   ```
3. **执行层处理**：执行脚本应负责将 SQL 映射到正确的数据库。如果执行环境要求指定 Schema，应在**表名**之前添加前缀（如 `dbname.table_name`），而不能在整个 SQL 语句前添加。
4. 脚本将生成 `result/测试用例.xlsx`，包含：序号、问句、执行SQL、执行结果。

## 注意事项
- **错误处理**：当 SQL 执行成功但未查到数据时，结果为 "查询成功，但未匹配到符合条件的数据"。当 SQL 执行过程中发生异常（如语法错误、字段不存在等）时，结果为具体的错误信息 `f"SQL执行出错: {str(e)}"`。当数据库连接失败时，结果为 `f"数据库连接失败: {str(e)}"`。
- **性能建议**：对于大数据量查询，建议在 SQL 中添加合理的限制条件。
- **安全**：数据库密码在 JSON 中以明文传输，请确保在安全环境下操作。
